# Kontava Dashboard Experience Execution Roadmap Skill Prompt

Using the document **"Kontava Dashboard Experience Innovation Run Report"** as the primary source, create a comprehensive, execution-ready roadmap for implementing the full dashboard experience innovation proposal across Kontava.

The goal is to transform the proposal from a strategic UX/product analysis into a precise implementation plan that can be executed step by step through Codex skills. By the end of the roadmap, the full proposal should be implementable across the Owner War Room, Manager Action Center, Cash Command, Analytics, Accounting, Assurance, and future BI command surfaces.

Analyze the report carefully and produce a complete execution plan that covers:

## 1. Proposal Extraction

Identify every major recommendation inside the report, including:

- Kontava Daily Command Environment.
- Command Brief Strip.
- What Changed Since Last Review.
- What Needs Action Today.
- Business Truth Zones.
- Proof Drawer and Evidence Timeline.
- Owner Morning Brief.
- Manager Daily Run Sheet.
- Cash Truth Map.
- Risk and Opportunity Radar.
- Evidence-Backed Insight Timeline.
- Stock-to-Cash Flow View.
- Close Readiness Journey.
- Daily habit loops.
- Shared BI command primitives.
- Technical/data prerequisites.
- Anti-bloat rules.
- Release gates.

For each recommendation, explain:

- What it is.
- Why it matters.
- Which current Kontava foundations it builds on.
- Which stakeholder it serves.
- Which product surfaces it affects.
- What must exist before it can be implemented safely.

## 2. Full Implementation Roadmap

Create a phased roadmap for fully implementing the proposal inside Kontava.

Organize the roadmap into precise phases such as:

1. BI Command UX Foundation.
2. Shared BI Contracts and Primitives.
3. Owner War Room Recomposition.
4. Manager Action Center Recomposition.
5. Cash Command Intelligence.
6. Analytics Repositioning into Business Pulse.
7. Accounting, Assurance, and Close Journey.
8. Proof Drawer and Evidence Timeline Expansion.
9. Daily Habit Loops and Digest System.
10. Full rollout, validation, performance, and release hardening.

For each phase, define:

- Objective.
- Business reason.
- Technical reason.
- User experience outcome.
- Exact files, services, components, actions, schemas, and tests likely involved.
- Dependencies.
- Step-by-step implementation sequence.
- What should be built first.
- What should be delayed.
- What must not be broken.
- Validation commands.
- Completion criteria.

## 3. Skill-Executable Step Design

Convert the roadmap into a suite of precise Codex-executable skills.

For each skill, define:

- Skill name.
- Purpose.
- Trigger condition.
- Source files and folders to inspect first.
- Allowed code areas.
- Forbidden code areas.
- Required implementation workflow.
- Required backend work.
- Required frontend work.
- Required UX behavior.
- Required security and compliance checks.
- Required tests.
- Required release gates.
- Expected deliverables.
- Completion criteria.
- Rollback considerations.

Each skill must be narrow enough to execute safely, but meaningful enough to produce real progress.

## 4. Recommended Skill Suite

Design a complete skill suite for executing the proposal, including skills such as:

- `kontava-bi-command-foundation`
- `kontava-bi-command-primitives`
- `kontava-owner-morning-brief`
- `kontava-manager-daily-run-sheet`
- `kontava-cash-truth-map`
- `kontava-business-pulse-analytics`
- `kontava-proof-evidence-timeline`
- `kontava-stock-to-cash-flow-view`
- `kontava-close-readiness-journey`
- `kontava-daily-habit-digest`
- `kontava-dashboard-release-gates`

For each skill, provide a full prompt that can be used to create and install the skill.

## 5. Technical Execution Details

For every phase and skill, specify required work across:

- `services/bi/**`
- `components/bi/**`
- `services/snapshots/**`
- `services/signals/**`
- `services/evidence/**`
- `services/security/**`
- `services/modules/**`
- `components/owner-war-room/**`
- `services/owner-war-room/**`
- `components/manager-action-center/**`
- `services/manager-action-center/**`
- `components/finance/**`
- `components/analytics/**`
- `components/accounting/**`
- `components/assurance/**`
- relevant server actions
- relevant tests
- `app/globals.css` only if shared dashboard primitives require safe styling additions

Preserve existing dashboard color semantics, BI trust states, tenant isolation, RBAC, module entitlements, redaction, auditability, proof trails, ledger-first accounting, OHADA compliance, and existing workflows.

## 6. UX And Product Execution Plan

Define how the user experience should evolve step by step.

Cover:

- How the first screen should change.
- How the user moves from brief to command to investigation mode.
- How proof drawers should appear.
- How actions should sit beside insights.
- How stale, partial, blocked, redacted, permission-denied, and module-unavailable states should appear.
- How owner, manager, accountant, finance, stockkeeper, payroll, and branch supervisor views should differ.
- How the dashboards become daily go-to surfaces rather than occasional reports.

## 7. Anti-Bloat And Safety Rules

For every roadmap step, clearly define:

- What must not be built yet.
- What should remain read-only first.
- What should reuse existing services and components.
- What should not become a disconnected mini-app.
- What would look impressive but add little SMB value.
- What should wait until data quality, usage, or customer demand proves the need.

## 8. Testing And Release Gates

Define a validation plan covering:

- Typecheck.
- Lint.
- Unit tests.
- Service tests.
- Component tests.
- Server action tests.
- RBAC and module entitlement tests.
- Redaction tests.
- Proof drawer tests.
- E2E tests.
- Mobile and desktop screenshot checks.
- Performance budget checks.
- Release gate script usage where available.

Each phase must include a clear "ready to proceed" gate.

## 9. Final Deliverables

Produce a professional execution blueprint containing:

- Executive summary.
- Proposal inventory.
- Full implementation roadmap.
- Skill suite list.
- Detailed prompt for each skill.
- Execution order.
- Dependency map.
- Technical work plan.
- UX work plan.
- Security and compliance plan.
- Anti-bloat rules.
- Testing and release gates.
- Risk register.
- Rollback strategy.
- Clear recommendation on which skill to create and execute first.

The final output should be practical, precise, enterprise-grade, and implementation-ready. It should make it possible to execute the entire Kontava Dashboard Experience Innovation proposal step by step through well-scoped Codex skills until the proposal is fully implemented across the platform.
