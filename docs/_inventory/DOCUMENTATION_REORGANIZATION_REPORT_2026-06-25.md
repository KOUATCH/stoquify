## Ambiguous Choices

- Historical `documentation/` files were placed in `archive/legacy-documentation/` because that folder mixed old product, RBAC, finance, POS, and navigation exports.
- Prompt-like files were separated into `prompts/` even when they describe implementation, so execution reports remain easier to distinguish from instructions for future agents.
- Generated report folders that still contain JSON baselines were not moved wholesale; only markdown documentation was moved, leaving machine-readable baselines in place where scripts may still expect them.
- Dot-prefixed skill staging folders, installed skill folders, and `graphify-out/` were treated as tool/runtime artifacts and left untouched.

## Link Handling

- Historical reports often quote old output paths and shell commands. Those were preserved to avoid falsifying past run evidence.
- Use `_inventory/documentation-move-map-2026-06-25.csv` to resolve old documentation paths after this cleanup.
