# Kontava Moat Execution Skill Blueprint Prompt

Using the **Kontava Moat Foundation Execution Roadmap** and the **Kontava Cross-Boundary Innovation and Moat Proposal** as source documents, conduct a deep technical, architectural, product, security, UX, and execution analysis to transform the roadmap into a practical suite of workable Codex execution skills.

The goal is to refine and architect the best implementation proposals for every baseline foundation Kontava needs before executing its cross-boundary innovation moat strategy. The final output must help the team move from strategic planning into a structured execution phase without destabilizing the existing platform.

Analyze the documents carefully and produce an execution-ready skill architecture that covers:

## 1. Foundation Breakdown

- Identify every baseline foundation required for the Kontava moat execution.
- Separate foundations into technical, architectural, data, security, compliance, UX, product, testing, migration, and operational categories.
- Explain why each foundation is necessary before advanced cross-module innovation begins.
- Identify dependencies between foundations.

## 2. Best Proposal Per Foundation

For each foundation, propose the strongest professional implementation approach:

- Objective.
- Business reason.
- Technical reason.
- Required schemas, services, hooks, server actions, APIs, UI surfaces, guards, jobs, reports, and audit trails.
- Security and compliance requirements.
- Risks if implemented incorrectly.
- Expected product/business impact.

## 3. Skill Suite Architecture

Convert the roadmap into a set of executable Codex skills that can guide the implementation phase.

For each proposed skill, define:

- Skill name.
- When the skill should be triggered.
- What problem it solves.
- Required source files or areas to inspect.
- Step-by-step workflow.
- Files or layers it is allowed to touch.
- What it must never break.
- Required validation commands.
- Expected deliverables.
- Completion criteria.

## 4. Recommended Skill Categories

Design skills for areas such as:

- Control plane, tenant isolation, RBAC, and module entitlements.
- Business event gateway and audit evidence.
- Ledger-first accounting backbone.
- Payment reconciliation and source linking.
- Inventory, POS, purchasing, payroll, and finance integration.
- Cross-module data trust and evidence graph foundations.
- Owner War Room readiness.
- Cash leakage and fraud-control foundations.
- Compliance readiness radar.
- UX shell, dashboard, alerting, and upgrade prompts.
- Migration, seed data, testing, release gates, and rollback.

## 5. Execution Sequencing

Create a logical order for building these skills:

- Must-build first.
- Low-risk foundation skills.
- Medium-risk consolidation skills.
- Advanced innovation skills that must wait.
- Skills that can run independently.
- Skills that must run in strict sequence.
- Validation gate before moving from one skill to the next.

## 6. Skill Design Quality

Ensure every skill is:

- Narrow enough to execute safely.
- Broad enough to produce meaningful progress.
- Clear about inspection before modification.
- Protective of tenant isolation, RBAC, auditability, ledger-first logic, OHADA compliance, and existing workflows.
- Designed for surgical code changes, not speculative rewrites.
- Verifiable through typecheck, lint, tests, route checks, service tests, E2E tests, and security checks where relevant.

## 7. Final Deliverable

Produce a complete **Kontava Moat Execution Skill Blueprint** containing:

- Executive summary.
- Foundation dependency map.
- Prioritized skill list.
- Detailed specification for each skill.
- Recommended skill folder names.
- Recommended SKILL.md frontmatter descriptions.
- Execution sequence.
- Validation and release gates.
- Risk register.
- Rollback strategy.
- Team responsibilities.
- Clear recommendation on which skill should be created and executed first.

The output must be practical, technical, enterprise-grade, and ready to convert into actual installed Codex skills. It should help Kontava begin the execution phase of the Cross-Boundary Innovation and Moat Proposal safely, professionally, and in the correct order.
