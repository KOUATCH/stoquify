### Foundation Suite Responsibilities

This suite is designed to ensure Kontava has the minimum safe platform backbone before building advanced product intelligence:

- Shared BI contracts for all KPI and proof-backed surfaces.
- Evidence grade and proof trail consistency.
- Snapshot/read-model contracts for fast, tenant-safe dashboards.
- Durable manager actions and action queue readiness.
- Module entitlement observe/enforcement readiness.
- Redaction and sensitive surface policy.
- Drill-through and proof drawer standards.
- Data quality, source-linking, and business event consistency.
- Seed, demo, and backfill readiness.
- Release gates, observability, rollback, and support diagnostics.
- Dashboard and BI UX primitives.
- Performance budgets and rebuild strategies.

### Foundation Suite Must Not Break

The suite prompts explicitly protect:

- tenant isolation
- RBAC
- module entitlements
- audit logs
- ledger-first accounting
- OHADA compliance semantics
- payment reconciliation
- inventory, POS, purchasing, payroll, finance, and compliance workflows
- existing analytics and Owner War Room routes
- existing tenant data and seeded scenarios

### Moat Build Suite Must Not Break

The suite prompts explicitly prohibit:

- bypassing server-side tenant/RBAC/module guards
- making client-side hiding the only protection
- exposing sensitive payroll, supplier bank, payment, close, partner, export, or proof-trail data without redaction
- posting inferred insight directly to the ledger
- introducing hard module enforcement before observe-mode gates pass
- building broad AI, partner APIs, or advanced evidence graph features before trusted source links, proof trails, and release gates are strong enough
- duplicating source module data into unmanaged BI tables
- turning Kontava into disconnected mini-apps
