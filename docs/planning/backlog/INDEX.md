# Backlog

Generated by pro-grade-audit on 2026-05-22.

**Project**: StockFlow — multi-tenant retail/POS SaaS. Next.js 15.1 + React 19 + Prisma 6 + Auth.js v4 (next-auth 4.24) + Postgres. ~50 paying orgs, 6 months live, pre-funding. Vercel-deployed. Sentry + Resend + Inngest wired. ~114 server actions across `actions/`, modernized domain logic in `services/`.

**Calibration**: solo developer shipping publicly, free-tier-only budget for any new SaaS line items, direct tone.

**Headline findings**:
1. Several server actions accept `organizationId` / `roleId` / `userId` directly from the client and trust them — cross-tenant account-takeover surface (tickets 001, 003, 006, 007, 008, 009).
2. Rate limiter is in-memory `Map`-based — broken on Vercel serverless/edge. Auth endpoints are effectively unthrottled (002).
3. ~107 of 114 server actions have no permission check — middleware-only gating means anyone authenticated can call any server action via fetch (004, 005).
4. CSP allows `unsafe-inline` + `unsafe-eval` — defeats CSP. `X-XSS-Protection` header is deprecated and should be removed (015).
5. No CI workflow despite the repo's expectations; deploy doesn't run `prisma migrate deploy` automatically (019, 022).
6. Permission notation has a silent-failure bug in `lib/permissions.ts`'s naive pluralizer (011).

## Recommended order (top 10 to tackle first)

1. **[002]** Upstash rate limiter — current limiter is effectively non-functional. 30 minutes of work; closes one of the most exploitable surfaces.
2. **[001]** Fix `createInvitedUser` (token-based redemption). Single largest takeover surface.
3. **[006]** Fix `getAllUsers` no-arg cross-tenant dump. PII leak; GDPR-relevant.
4. **[007]** `/api/v1/organisations/[id]/*` org-boundary checks. Same root cause, smaller blast radius.
5. **[009]** Fix admin-role lookup in new-org signup. Subtle but real.
6. **[008]** POS cross-org session trust — money path; do before signups grow.
7. **[004]** `protect()` wrapper. Once it exists, future server actions cost ~10 lines of correct authz instead of 50 lines of optional authz.
8. **[005]** Zod validation at boundaries. Pairs with `protect()`; close input-trust holes the moment the wrapper is in place.
9. **[003]** Sweep the long-tail of client-trusted-ID server actions. Tedious but mechanical once #004 + #005 are in place.
10. **[014]** Org-scoped Prisma extension. Turns future cross-tenant bugs into impossibilities at the query layer.

After that: the testing pair (017 + 018) locks the foundation in place, 019 (CI) keeps it from drifting, 020 (Sentry tags) gives you incident-response superpowers, and 011/012/013/015/021/022 are all S-effort housekeeping that can land in parallel with the bigger items.
