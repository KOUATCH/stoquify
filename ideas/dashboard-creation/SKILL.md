# Skill: dashboard-creation

## Purpose

Create exceptional, enterprise-grade dashboard UI concepts for SaaS and business operations products. Use this skill when designing dashboards, command centers, analytics pages, admin consoles, ERP interfaces, POS back offices, finance dashboards, workforce dashboards, inventory dashboards, or any data-rich product UI.

The skill turns product requirements into a polished dashboard design brief inspired by world-class SaaS and enterprise software dashboards.

## Triggers

Use this skill when the user asks for:

- Dashboard design inspiration.
- Enterprise dashboard creation.
- Admin panel UI direction.
- SaaS dashboard UX.
- ERP dashboard design.
- POS, inventory, finance, payroll, attendance, purchasing, or reporting dashboard UI.
- A premium command center concept.
- Modern product UI examples.
- “Stunning”, “professional”, “enterprise-grade”, “robust”, or “high-quality” dashboard design.

Example prompts:

```text
Create a dashboard design direction for my ERP.
```

```text
Research the best enterprise dashboards and design one for my product.
```

```text
Make a modern command center for inventory, POS, payroll, and finance.
```

```text
Design a stunning admin dashboard for a business operations platform.
```

## Target Product Context

Default product context, unless the user says otherwise:

```text
A unified business operations platform covering POS, inventory, purchasing, finance, payroll, attendance, reporting, automation, branches, permissions,receivables, payables and administration.
```

The dashboard should feel:

- Stunning and premium.
- Professional and enterprise-grade.
- Modern, clean, and visually beautiful.
- Robust enough for serious business use.
- Data-rich without feeling cluttered.
- Built with recent UI/UX patterns and technology.
- Suitable for owners, managers, cashiers, accountants, stock controllers, HR managers, and admins.

## Research Checklist

For each selected dashboard, analyze:

1. Product name and URL.
2. What the dashboard helps users do.
3. Overall visual style.
4. Layout and information architecture.
5. Sidebar/menu structure.
6. Cards, tables, charts, and data visualization patterns.
7. Color, spacing, typography, density, and hierarchy.
8. Interactions, motion, filters, command palettes, AI features, and automation.
9. Enterprise trust signals: permissions, audit logs, role-based access, reliability, security, compliance, or operational controls.
10. Ideas to adapt for the target product.
11. What not to copy directly.

## Dashboard Creation Workflow

### 1. Clarify Product Role

Identify the dashboard’s primary user and job-to-be-done.

If the user has not specified a role, design for the owner/operator first and include variations for:

- Owner
- Cashier
- Stock manager
- Accountant
- HR manager
- Admin

### 2. Define Command Center Promise

Write a one-sentence dashboard promise.

Strong default:

```text
See sales, stock, cash, purchases, payroll, attendance, accounts payables and accounts receivables and approvals in one live command center.
```

### 3. Select Inspiration Set

Choose five relevant references from the benchmark list.

Recommended default set for unified business operations:

- Linear for elegance and sidebar discipline.
- Stripe for finance-grade clarity.
- Rippling for unified employee, finance, and automation data.
- Shopify Admin or Square for POS and commerce operations.
- Odoo or NetSuite for ERP breadth and enterprise depth.

### 4. Synthesize Design Direction

Produce a dashboard design direction covering:

- Overall layout.
- Sidebar/navigation.
- Homepage command center.
- Cards.
- Tables.
- Charts.
- Filters.
- Search.
- Role-based variations.
- Enterprise controls.
- Mobile/responsive behavior.

### 5. Design The Main Dashboard

Describe the actual dashboard screen with sections.

Default command center structure:

```text
Top bar:
- Global search
- Branch switcher
- Date range
- Quick create
- Notifications
- User menu
- Language switcher

Hero summary row:
- Today’s sales
- Gross margin
- Cash on hand
- Low-stock items
- Staff present
- Pending approvals

Main grid:
- Sales trend
- Inventory risk
- Cash flow snapshot
- Purchase pipeline
- Attendance live view
- Payroll upcoming
- Alerts and exceptions
- Recent activity

Operational tables:
- Low stock
- Pending purchase approvals
- Unpaid bills
- Late/absent employees
- Recent POS transactions
```

### 6. Add Role-Based Variants

Generate role-specific dashboards:

Owner:

```text
Revenue, profit, cash, stock risk, payroll cost, branch performance, approvals.
```

Cashier:

```text
Open POS, active shift, today’s sales, receipts, returns, customer lookup.
```

Stock manager:

```text
Low stock, transfers, stock movement, purchase requests, goods received.
```

Accountant:

```text
Receivables, payables, expenses, payroll cost, cash flow, audit trail.
```

HR manager:

```text
Attendance, shifts, leave, overtime, salary runs, payslips.
```

Admin:

```text
Users, roles, permissions, devices, branches, imports, audit logs, settings.
```

### 7. Specify Premium UI Details

Include UI details that make the dashboard feel high-end:

- Calm density with expandable detail.
- Strong typographic hierarchy.
- Status colors used only for meaning.
- Rounded cards with subtle borders.
- Data tables with sticky headers and saved views.
- Chart tooltips with exact values.
- Empty states that teach users what to do.
- Inline audit indicators for sensitive actions.
- Command palette for power users.
- Contextual quick actions.

## Design Principles

Use these principles for every dashboard concept:

```text
Clarity before decoration.
Density without clutter.
Operational truth in real time.
Role-aware by default.
Financial data must feel trustworthy.
Tables must be powerful, not plain.
Charts must answer business questions, not just look pretty.
Every alert should imply an action.
Every module should feel connected.
```

## Suggested Dashboard Aesthetic

Default palette:

```text
Primary ink: #0B1220
Deep navy: #123B66
Electric blue: #2563EB
Operational green: #10B981
Stock warning: #F59E0B
Critical red: #EF4444
Soft background: #F6F8FB
Card background: #FFFFFF
Card border: #E2E8F0
Muted text: #64748B
```

Default visual direction:

```text
Dark icon rail.
Light workspace.
Data cards with subtle borders.
Charts with restrained color.
Tables with excellent filtering.
Command bar always available.
Status badges for operational urgency.
```

## Anti-Patterns

Avoid:

- Generic admin templates.
- Random gradient cards with no meaning.
- Too many colors.
- Dashboard cards that only show vanity metrics.
- Dense ERP sidebars with every feature visible at once.
- Charts without decisions attached.
- Hiding critical alerts below the fold.
- Giving all roles the same dashboard.
- Making finance screens feel playful.
- Making POS workflows feel slow or overdesigned.

## Success Criteria

A successful dashboard concept should make the product feel:

```text
The clarity of a dashboard.
The speed of a POS.
The discipline of finance software.
The automation of workforce management.
The control of an ERP.
```

The user should immediately understand:

```text
What is happening.
What needs attention.
What changed recently.
What action to take next.
```
