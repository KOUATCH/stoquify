## Decision

Status: implemented and evidence-gated for the current service-owned calculation slice.

This closes a material payroll-engine hardening gap for controlled progression toward full HR/payroll production: correction payroll runs can now reference a posted or paid original run, calculate scoped employee deltas in a separate correction period, preserve original register proof hashes, and emit correction evidence without mutating the original payroll register.

This does not make the full HR/payroll module production-ready by itself. It is one Wave 1 blocker reduction under the statutory/payroll-engine hardening lane.

## Production Meaning

The payroll engine is now safer for retro/correction behavior because it records correction deltas as new immutable payroll run facts instead of reopening or mutating the original register. Accounting, declarations, payments, close assurance, and BI can consume the correction run as a separate financial event with explicit original-run evidence.

This supports the roadmap spine:

HR source data -> contracts/compensation/attendance -> payroll run -> immutable payroll register -> payslips, ledger, payments, declarations -> close assurance/data trust -> finance/BI/cash planning.

## Residual Risks

- Country-pack statutory breadth is still not complete for unrestricted production across jurisdictions.
- Correction tests currently cover one core delta path; broader negative correction, missing original-line, IRPP-enabled, rubrique-change, overtime-change, and multi-employee correction fixtures should be added before full production signoff.
- Operator UI should not expose broad correction actions until service-backed review/approval, evidence drawers, fresh auth, and denied/redacted states are wired.
- Accounting posting tests should add explicit correction-run posting assertions so correction deltas flow through ledger mappings as separate financial events.
- Migration/backfill still needs tenant-by-tenant dry run, rollback/correction strategy, signoff workflow, and reconciliation proof.

## Next Best Step

Continue Wave 1 with broader correction fixtures and accounting tie-out:

1. Add negative adjustment fixtures for salary decrease, deduction increase, benefit/rubrique removal, overtime reversal, and IRPP-enabled correction deltas.
